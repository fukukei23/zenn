## はじめに

Gitを利用した開発において、`.gitignore` は不可欠なファイルです。ビルド成果物や一時ファイル、環境設定ファイルなどをGitの管理対象から外すために、誰もが毎日のように記述していることでしょう。

しかし、この `.gitignore` の書き方を少しでも間違えると、「コミットしたはずのファイルがリポジトリに存在しない」という厄介な落とし穴に陥ります。問題に気づくのが遅れると、他の開発者のローカル環境でアプリが動かなくなるなどの重大なトラブルに発展しかねません。

本記事では、私が実際に開発に携わっているプロジェクト（NexusCore）で発生した `.gitignore` にまつわるトラブルを題材に、「親ディレクトリのつもりで指定した名前が、意図せず子ディレクトリまで巻き込んで除外されてしまう問題」の発生原因と修正手順を解説します。

## 何が起きたのか？：不可解な「ファイルの行方不明」

NexusCoreというプロジェクトでは、バックエンドのAPI処理にPythonとFastAPIを利用し、さらに一部の管理画面やフロントエンドの資産を `src/nexuscore/webapp/` というディレクトリで管理しています。

ある日、ルートディレクトリにビルド済みのフロントエンド成果物を格納するための `webapp/` というディレクトリを新設し、このルート直下のディレクトリだけをGitの管理対象から外すことにしました。

開発者は、何気なく `.gitignore` の末尾に以下の1行を追記しました。

```gitignore:.gitignore
# ビルド成果物を除外
webapp/
```

一見すると正しい記述に見えます。しかし、その後の開発で `src/nexuscore/webapp/` 配下に新しいPythonのモジュールやテンプレートを追加し、コミットしてプッシュしたところ、なぜかそのファイル群がGitHub上のリポジトリに一切反映されていないことが発覚したのです。

## なぜこの罠にハマるのか（.gitignoreの仕様）

なぜルートの `webapp/` を除外しただけなのに、深い階層にある `src/nexuscore/webapp/` まで消えてしまったのでしょうか？

原因は、**Gitの `.gitignore` におけるパターンマッチングの仕様**にあります。

`.gitignore` に `webapp/` とだけ記述した場合、Gitはこれを「**ディレクトリ名が `webapp` であるものすべて**」という解釈をします。つまり、パスの階層（ルートであるか、サブディレクトリであるか）を問わず、プロジェクト内のあらゆる場所にある `webapp` という名前のディレクトリをGitの追跡対象から外してしまうのです。

これが「包含漏れ」と呼ばれるトラブルの根本原因です。初心者あるあるな罠ですが、気づきにくいため非常に厄介です。

## 修正手順とPythonプロジェクトにおける注意点

### 1. .gitignoreの修正

この問題を解決するには、Gitに「**ルートディレクトリ直下にある `webapp/` だけ**」を明示的に伝える必要があります。それには、パスの先頭にスラッシュ `/` を付与します。

修正前と修正後の `.gitignore` は以下のようになります。

```gitignore:.gitignore
# ❌ 修正前：すべての階層の webapp ディレクトリが除外されてしまう
webapp/

# ✅ 修正後：ルートディレクトリ直下の webapp ディレクトリのみを除外する
/webapp/
```

このように先頭に `/` をつけることで、任意のサブディレクトリにある同名ディレクトリを巻き込むことを防げます。

### 2. Python（FastAPI）における影響と具体例

NexusCoreではPythonを利用しているため、もし重要な設定ファイルやアセットが巻き込まれて除外されると、アプリケーションの起動すらできなくなります。

例えば、FastAPIで静的ファイルやテンプレートをマウントする以下のようなPythonコードがあったとします。

```python:src/nexuscore/main.py
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
import os

app = FastAPI()

# src/nexuscore/webapp/ 配下の静的ファイルを公開する
webapp_dir = os.path.join(os.path.dirname(__file__), "webapp")

# もし .gitignore の誤記により webapp/ が丸ごとGit管理外になっていた場合、
# 他の開発者がクローンした際にこのディレクトリが存在せずエラーとなる
if os.path.exists(webapp_dir):
    app.mount("/static", StaticFiles(directory=webapp_dir), name="static")
else:
    # 本来存在すべきディレクトリがない場合のフォールバック
    print("Warning: webapp directory is missing!")
```

一見するとコードに問題はないですが、Gitからファイルが欠落していると、他の開発者がリポジトリをクローンした直後に `directory is missing!` という警告が出力され、画面が真っ白になるといった現象に見舞われます。

### 3. すでに除外されてしまったファイルを復活させる

`.gitignore` を修正してコミットするだけでは、すでにGitの追跡対象から外れてしまったファイルは復活しません。以下の手順でキャッシュをクリアし、再度追跡対象に含める必要があります。

```bash
# Gitのインデックス（キャッシュ）から該当ディレクトリを削除（実ファイルは消えません）
git rm -r --cached src/nexuscore/webapp/

# .gitignore を修正した上で、再度追加
git add .gitignore
git add src/nexuscore/webapp/

# コミットしてプッシュ
git commit -m "fix: .gitignoreのwebapp/がsrc/nexuscore/webapp/を巻き込む不具合を解消"
git push origin main
```

実際のNexusCoreプロジェクトでも、このキャッシュ削除と `.gitignore` の修正（`/webapp/` への変更）を行うことで、無事にファイルがリポジトリに反映されるようになりました。

## おわりに

`.gitignore` の記述ルールはシンプルですが、ゆえに思いがけない大事故を引き起こすことがあります。特にディレクトリ名だけで除外指定をした場合は、プロジェクトのどこかに同名のディレクトリがないかを注意深く確認する癖をつけることが重要です。

今回のポイントをまとめます。

- 特定の階層のディレクトリだけを除外したい場合は、必ず先頭に `/` （例： `/webapp/`）をつける
- 除外設定を追加した後は、意図したファイルのみが除外されているか `git status` でこまめに確認する
- 「あれ？コミットしたはずのファイルがない」という時は、`.gitignore` の巻き込み漏れを疑う

同様の問題で頭を抱えている方や、これからGitを学び始める初心者の方のトラブルシューティングとして、本記事が少しでもお役に立てば幸いです。
